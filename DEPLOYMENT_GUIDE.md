## 🚀 OPTION 1: Deploy to Render.com (EASIEST - RECOMMENDED)

### Step 1: Prepare Your Application

Create `render.yaml` in your project root:

```yaml
services:
  - type: web
    name: nutrition-diet-expert
    env: python
    region: oregon
    plan: free
    buildCommand: pip install -r requirements.txt
    startCommand: gunicorn run:app
    envVars:
      - key: PYTHON_VERSION
        value: 3.11.9
      - key: SECRET_KEY
        generateValue: true
      - key: SKIP_DB_CREATE_ALL
        value: 0

databases:
  - name: nutrition-db
    databaseName: nutrition_diet_db
    plan: free
```

### Step 2: Add Gunicorn to requirements.txt

Add this line to `requirements.txt`:
```
gunicorn==21.2.0
```

### Step 3: Push to GitHub

```bash
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/YOUR_USERNAME/YOUR_REPO.git
git push -u origin main
```

### Step 4: Deploy on Render

1. Go to https://render.com
2. Sign up (free)
3. Click "New +" → "Web Service"
4. Connect your GitHub repository
5. Render will auto-detect `render.yaml`
6. Click "Create Web Service"
7. Wait 5-10 minutes for deployment
8. Your app will be live at: `https://your-app-name.onrender.com`

### Step 5: Set Environment Variables (in Render Dashboard)

```
DATABASE_URL=<automatically set by Render if you created database>
SECRET_KEY=<auto-generated or set manually>
SKIP_DB_CREATE_ALL=0
```

### Step 6: Run Database Seeds

Go to Render Shell and run:
```bash
python seeds/seed.py restore
```

## 🚀 OPTION 2: Deploy to Railway.app

### Step 1: Install Railway CLI (Optional)

```bash
npm install -g @railway/cli
```

Or deploy via web interface (easier).

### Step 2: Prepare Application

Create `Procfile` in project root:
```
web: gunicorn run:app
```

Create `railway.json`:
```json
{
  "build": {
    "builder": "NIXPACKS"
  },
  "deploy": {
    "startCommand": "gunicorn run:app",
    "restartPolicyType": "ON_FAILURE"
  }
}
```

### Step 3: Deploy

1. Go to https://railway.app
2. Sign up with GitHub
3. Click "New Project"
4. Click "Deploy from GitHub repo"
5. Select your repository
6. Railway auto-deploys!

### Step 4: Add PostgreSQL Database

1. In your project, click "New"
2. Select "Database" → "Add PostgreSQL"
3. Railway will automatically set `DATABASE_URL`

### Step 5: Run Seeds

Use Railway CLI or web shell:
```bash
railway run python seeds/seed.py restore
```

## 🚀 OPTION 3: Deploy to PythonAnywhere

### Step 1: Sign Up

1. Go to https://www.pythonanywhere.com
2. Create free "Beginner" account

### Step 2: Upload Your Code

Option A: Use Git
```bash
git clone https://github.com/YOUR_USERNAME/YOUR_REPO.git
```

Option B: Upload files via web interface

### Step 3: Set Up Virtual Environment

In PythonAnywhere Bash console:
```bash
cd ~/nutrition_diet_expert_system-main
python3.11 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

### Step 4: Configure Web App

1. Go to "Web" tab
2. Click "Add a new web app"
3. Choose "Flask"
4. Set Python version to 3.11
5. Set source code directory
6. Configure WSGI file (see below)

### Step 5: Edit WSGI Configuration

In `/var/www/yourusername_pythonanywhere_com_wsgi.py`:
```python
import sys
import os

# Add your project directory
project_home = '/home/yourusername/nutrition_diet_expert_system-main'
if project_home not in sys.path:
    sys.path.insert(0, project_home)

# Load environment variables
from dotenv import load_dotenv
load_dotenv(os.path.join(project_home, '.env'))

# Import your Flask app
from run import app as application
```

### Step 6: Set Up Database

PythonAnywhere provides MySQL. Update `.env`:
```
DB_HOST=yourusername.mysql.pythonanywhere-services.com
DB_USER=yourusername
DB_PASSWORD=<your-mysql-password>
DB_NAME=yourusername$nutrition_db
```

### Step 7: Run Seeds

In Bash console:
```bash
cd ~/nutrition_diet_expert_system-main
source venv/bin/activate
python seeds/seed.py restore
```

## 🚀 OPTION 4: Deploy to Vercel (Already Configured!)

Your code already has Vercel support! Look at the `IS_VERCEL` checks in `config.py`.

### Step 1: Install Vercel CLI

```bash
npm install -g vercel
```

### Step 2: Create `vercel.json`

```json
{
  "version": 2,
  "builds": [
    {
      "src": "run.py",
      "use": "@vercel/python"
    }
  ],
  "routes": [
    {
      "src": "/(.*)",
      "dest": "run.py"
    }
  ],
  "env": {
    "VERCEL": "1",
    "SKIP_DB_CREATE_ALL": "1"
  }
}
```

### Step 3: Set Up Supabase (Free PostgreSQL)

1. Go to https://supabase.com
2. Create free account
3. Create new project
4. Get connection string from Settings → Database
5. Add to Vercel environment variables

### Step 4: Deploy

```bash
vercel login
vercel
```

Or connect GitHub repo in Vercel dashboard.

### Step 5: Set Environment Variables in Vercel

```
SUPABASE_DB_URL=postgresql://...
SECRET_KEY=your-secret-key
SKIP_DB_CREATE_ALL=1
```

## 🚀 OPTION 5: Deploy to Fly.io

### Step 1: Install Fly CLI

Windows:
```powershell
powershell -Command "iwr https://fly.io/install.ps1 -useb | iex"
```

### Step 2: Login and Initialize

```bash
fly auth login
fly launch
```

### Step 3: Create `fly.toml`

Fly will generate this, but customize:
```toml
app = "nutrition-diet-expert"

[build]
  builder = "paketobuildpacks/builder:base"

[env]
  PORT = "8080"
  SKIP_DB_CREATE_ALL = "0"

[[services]]
  http_checks = []
  internal_port = 8080
  protocol = "tcp"
  
  [[services.ports]]
    force_https = true
    handlers = ["http"]
    port = 80
  
  [[services.ports]]
    handlers = ["tls", "http"]
    port = 443
```

### Step 4: Add PostgreSQL

```bash
fly postgres create
fly postgres attach <postgres-app-name>
```

### Step 5: Deploy

```bash
fly deploy
```

## 📋 Preparation Checklist for All Platforms

### 1. Update `requirements.txt`

Add these if missing:
```
gunicorn==21.2.0
psycopg2-binary==2.9.9  # For PostgreSQL
```

### 2. Create `.gitignore`

```
__pycache__/
*.pyc
*.pyo
*.pyd
.Python
env/
venv/
.env
*.db
*.sqlite3
instance/
.vscode/
.idea/
*.log
```

### 3. Update `.env` for Production

Create `.env.production` template:
```
SECRET_KEY=<use-strong-random-key>
DATABASE_URL=postgresql://user:pass@host:5432/dbname
SKIP_DB_CREATE_ALL=1
SESSION_COOKIE_SECURE=1
REMEMBER_COOKIE_SECURE=1
```

### 4. Create `runtime.txt` (for some platforms)

```
python-3.11.9
```

## 🔒 Security Checklist

Before deploying:

- [ ] Change SECRET_KEY (never use default!)
- [ ] Enable HTTPS (most platforms do this automatically)
- [ ] Set SESSION_COOKIE_SECURE=1
- [ ] Set REMEMBER_COOKIE_SECURE=1
- [ ] Change default passwords in database
- [ ] Review CORS settings if using API
- [ ] Enable rate limiting for production
- [ ] Set up database backups
